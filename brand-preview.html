Implementation — How Splash Screens Work & What to Do Next

How Expo Splash Screens Work

The OS holds the native splash on screen while your JavaScript bundle loads. expo-splash-screen lets you hold it longer and dismiss with a fade. The backgroundColor in app.config.ts fills any screen area not covered by the splash image — it must match the first screen's background color or the transition will show a jarring color jump. All SafeScan apps use #FDFBF7 (warm cream) as both splash bg and first-screen bg.

Minimum Fix — Already Applied

app.config.ts for ADHDScan has been updated: backgroundColor: "#8B5CF6". This alone eliminates the white flash. The splash image is still the placeholder, but the background color fills the screen in brand violet until the new PNG is in place. Requires eas build — JS-only OTA won't change native splash.

Exporting Icons to PNG

Option 1 — Figma: Paste SVG code into Figma frame, resize to 1024×1024, export PNG.

Option 2 — Browser: Right-click any icon above, "Save as SVG" (via DevTools), then convert.

Option 3 — CLI: Install sharp-cli and run:
npx sharp-cli -i icon.svg -o icon.png resize 1024

App Store Connect — Icon Update

The ADHDScan App Store listing currently shows the Expo default "A" icon. To update: export the violet lightning bolt icon at 1024×1024 PNG → App Store Connect → App Information → upload new icon. Icon-only changes can go through review without a full binary resubmit in some cases, but typically require a new build.

Splash Image Replacement

Replace assets/splash-icon.png with the exported PNG at 1284×2778px. The image should show the brand-color background with the white symbol centered — so it's visually identical to the backgroundColor shown behind it. Use resizeMode: "contain" so no cropping occurs on smaller devices.

Android Adaptive Icons

Android needs two separate files: android-icon-foreground.png (white symbol on transparent, 108×108dp safe zone) and android-icon-background.png (solid brand color). The foreground is just the white lightning bolt SVG on a transparent canvas. Background: solid #8b5cf6 fill.

Claude Reference — Asset Specs & Code Patterns for All 8 Apps

How SafeScan Systems Apps Are Built

Every app in the portfolio is a React Native / Expo (SDK 55+) app built from the same template as ThyraScan. Each app has a unique brand color and a medical symbol as its icon. The splash screen is a full-gradient background matching the icon, with the white symbol centered, the app name, tagline, and "by SafeScan Systems" byline. All asset PNGs are generated by scripts/generate-app-icon.js using sharp (not installed in package.json — run with npm install sharp --no-save). Splash transitions use expo-splash-screen with preventAutoHideAsync() and a 400ms fade, held until auth + subscription + profile providers all finish loading.

Required Asset Files (every app)

assets/icon.png — 1024×1024, no transparency, no rounded corners (App Store adds them)
assets/splash-icon.png — 1284×2778 (iPhone 14 Pro Max native), gradient bg + white symbol + text
assets/android-icon-foreground.png — 1024×1024, transparent bg, white symbol, symbol within 72% safe zone (~737px)
assets/android-icon-background.png — 1024×1024, solid brand color fill
assets/android-icon-monochrome.png — 1024×1024, black bg, white symbol

Generate all with:
npm install sharp --no-save
node scripts/generate-app-icon.js

app.config.ts Splash Settings

splash.image: "./assets/splash-icon.png"
splash.resizeMode: "contain"
splash.backgroundColor: Match the app's first screen background (#FDFBF7 for warm-cream apps) — this fills the area outside the image on smaller devices, and is visible briefly as the splash fades out. Must match or the transition is jarring.

android.adaptiveIcon.backgroundColor: Solid mid-point of gradient (e.g. #14b8a6 for ThyraScan). This is what shows behind the foreground on older Android.

Requires eas build to take effect — not patchable via OTA JS update.

Splash Script Structure (generate-app-icon.js)

Use the ThyraScan scripts/generate-app-icon.js as the template.

Key parameters to change per app:
1. thyroidPaths() → replace with the app's SVG paths function
2. Gradient stops in <linearGradient id="bg">
3. Android background fill color
4. App name string in the splash text element
5. Tagline string (e.g. "IBS-aware food scanning")

Visual center of symbol in 100-unit space:
→ ThyraScan: (50, 54.5)
→ All others: (50, 50) unless noted above

App.tsx — Controlled Splash Fade Pattern (copy this for every new app)

// 1. Install: npx expo install expo-splash-screen // 2. Top of App.tsx (module level — outside any component): import * as SplashScreen from "expo-splash-screen"; SplashScreen.preventAutoHideAsync(); SplashScreen.setOptions({ fade: true, duration: 400 }); // 3. Replace AppNavigator with a wrapper that hides after all providers load: function AppNavigatorWithSplash() { const { loading: authLoading } = useAuth(); const { loading: subscriptionLoading } = useSubscription(); const { loading: profileLoading } = useUserProfile(); const appReady = !authLoading && !subscriptionLoading && !profileLoading; useEffect(() => { if (appReady) SplashScreen.hideAsync(); }, [appReady]); return <AppNavigator />; } // 4. Inside the provider tree in App(), use AppNavigatorWithSplash instead of AppNavigator. // IMPORTANT: AppNavigatorWithSplash must be INSIDE all three providers so hooks work.
